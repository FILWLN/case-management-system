## ✅ 已确认事项

### 1. 执恢业务流程（Q28-Q30）
**确认结果：人工点击触发**
- 执恢由人工点击触发，非自动
- 执恢案件作为**独立案件**管理
- 通过 `parentCaseId` 关联原执行案件
- 历史记录以**列表形式**展示

**实现方案**：
```typescript
// 执恢案件
{
  id: "执恢-001",
  caseType: "执恢",
  parentCaseId: "执行-001", // 关联原执行案件
  pinCode: "JD123456",
  // ...其他字段
}
```

### 2. 状态流转规则（Q38-Q40）
**确认结果：人工点击变更**
- 状态变更由用户**手动点击**按钮触发
- 允许跳过中间状态（管理员权限）
- 每次状态变更需**记录操作人和时间**

**状态流转权限**：
| 操作 | 普通用户 | 管理员 | 调解员 |
|------|---------|--------|--------|
| 正常状态流转 | ✅ | ✅ | ✅ |
| 跳过状态 | ❌ | ✅ | ❌ |
| 回退状态 | ❌ | ✅ | ❌ |
