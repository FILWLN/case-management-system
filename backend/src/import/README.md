# 还款数据导入功能说明

## 功能概述

基于客户提供的 `还款明细_20260302-测试.xlsx` 文件，实现还款数据导入功能。

## 核心功能

### 1. 还款类型自动识别

根据**还款方式**自动判断：

| 还款方式关键词 | 识别结果 | 说明 |
|----------------|----------|------|
| 法院、划扣、扣划、强制执行、司法扣划 | 法院划扣 | 法院强制执行扣款 |
| 主动、线下、线上、转账、支付宝、微信、银行 | 主动还款 | 当事人主动还款 |
| 其他 | 主动还款（默认） | - |

**客户需求**：Q27 - 当事人主动还款与法院划扣需要区分，两项为不同还款路径，要分别列示

### 2. 案件关联

通过 **客户ID** 关联案件：
- 格式提取：jd_xxx → 提取 xxx 作为 PIN码
- 优先按 PIN码 查找案件
- 其次按 客户ID 查找案件
- 未找到案件时，记录错误

### 3. 金额处理

- **还款金额**：累加到 `amountInfo.paidTotal`
- **法院划扣**：累加到 `amountInfo.courtDeduction`
- **主动还款**：累加到 `amountInfo.voluntaryRepayment`
- **剩余金额**：自动重新计算

### 4. 数据验证

必填字段：
- ✅ 客户ID
- ✅ 还款金额（必须 > 0）
- ✅ 还款时间

## API 接口

### 1. 预览导入数据

```http
POST /api/import/repayments/preview
Content-Type: multipart/form-data

file: [Excel文件]
```

响应：
```json
{
  "success": true,
  "data": {
    "total": 19,
    "preview": [
      {
        "row": 2,
        "customerId": "jd_6bc609182f74d",
        "repaymentAmount": 2199.0,
        "repaymentDate": "2025-09-25",
        "repaymentType": "主动还款",
        "exists": true
      }
    ]
  }
}
```

### 2. 执行导入

```http
POST /api/import/repayments
Content-Type: multipart/form-data

file: [Excel文件]
operator: "管理员"
```

响应：
```json
{
  "success": true,
  "message": "导入完成：成功 19 条，失败 0 条",
  "data": {
    "total": 19,
    "success": 19,
    "failed": 0,
    "errors": [],
    "importedRecords": [...]
  }
}
```

### 3. 下载导入模板

```http
GET /api/import/repayments/template
```

返回：Excel模板文件

---

## 使用流程

1. **下载模板** → 获取标准导入格式
2. **填写数据** → 按模板格式填写还款数据
3. **预览检查** → 上传文件预览，确认数据正确
4. **执行导入** → 确认无误后执行导入
5. **查看结果** → 查看导入成功/失败记录

---

## 错误处理

| 错误类型 | 说明 | 处理方式 |
|----------|------|----------|
| 客户ID为空 | 必填字段缺失 | 记录错误，跳过该行 |
| 还款金额无效 | 金额 <= 0 | 记录错误，跳过该行 |
| 日期格式错误 | 无法解析日期 | 记录错误，跳过该行 |
| 未找到案件 | 客户ID无对应案件 | 记录错误，跳过该行 |
