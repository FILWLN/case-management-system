# 案管系统 - 基于客户确认清单的修改方案

## 一、现有系统能力评估

### ✅ 已支持的功能
| 功能 | 现有实现 | 满足度 |
|------|---------|--------|
| 案件基础管理 | Case实体 + Cases服务 | 60% |
| 还款计划生成 | RepaymentPlanGeneratorService | 40% |
| 资产包管理 | AssetPackage模块 | 70% |
| 用户认证 | Auth模块 | 80% |
| 通知系统 | Notifications模块 | 50% |

### ❌ 缺失的核心功能
1. **京东PIN码关联体系**（当前使用案件ID）
2. **民初案件 vs 执行案件** 区分管理
3. **法官信息模块**（承办法官、助理、书记员）
4. **冻结管理**（标记无需冻结场景）
5. **状态机引擎**（复杂的案件状态流转）
6. **Excel数据导入**（还款数据、案件数据）
7. **调解员角色**（当前只有管理员/用户）
8. **报表系统**（周报/月报）

## 三、数据库迁移脚本

```sql
-- 1. 案件表新增字段
ALTER TABLE cases ADD COLUMN pin_code VARCHAR(255) UNIQUE;
ALTER TABLE cases ADD COLUMN case_type VARCHAR(50);
ALTER TABLE cases ADD COLUMN primary_case_id VARCHAR(255);
ALTER TABLE cases ADD COLUMN judge_info JSON;
ALTER TABLE cases ADD COLUMN freeze_status VARCHAR(50) DEFAULT 'normal';
ALTER TABLE cases ADD COLUMN no_freeze_reason VARCHAR(255);
ALTER TABLE cases ADD COLUMN amount_info JSON;

-- 2. 创建法院表
CREATE TABLE courts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(255) NOT NULL,
  address VARCHAR(500),
  contact_phone VARCHAR(50),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 3. 创建法官表
CREATE TABLE judges (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(100) NOT NULL,
  role VARCHAR(50) NOT NULL,
  court_id UUID REFERENCES courts(id),
  phone VARCHAR(50),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 4. 创建数据导入日志表
CREATE TABLE import_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  import_type VARCHAR(50) NOT NULL,
  file_name VARCHAR(255),
  total_rows INTEGER,
  success_rows INTEGER,
  fail_rows INTEGER,
  error_details JSON,
  imported_by UUID,
  imported_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
