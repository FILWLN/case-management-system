## 一、期数和还款日配置

### 功能描述
- 在案件录入时，可配置"期数"和"还款日"
- 系统自动生成对应期数的还款计划表格
- 支持等额本息、等额本金、一次性还本付息等方式

### 数据模型
```typescript
// 还款计划配置
interface RepaymentConfig {
  totalPeriods: number;        // 总期数
  repaymentDay: number;        // 每月还款日 (1-31)
  repaymentMethod: string;     // 还款方式: equal_interest/equal_principal/once
  firstDueDate: Date;          // 首次还款日
  totalAmount: number;         // 总应还金额
}

// 自动生成还款计划
interface GeneratedInstallment {
  periodNo: number;            // 期号
  dueDate: Date;               // 应还日期
  dueAmount: number;           // 应还金额
  principal: number;           // 本金
  interest: number;            // 利息
  remainingPrincipal: number;  // 剩余本金
}
```

## 二、还款日录入提醒（站内信）

### 功能描述
- 系统每日自动检查次日/当日需要还款的案件
- 生成站内信提醒给相关操作人员
- 站内信包含：案件列表、应还金额、跳转链接

### 提醒规则
```
提醒时间: 每天 9:00
提醒内容: 
  - 今日应还案件
  - 明日应还案件（提前提醒）
  - 已逾期案件

站内信格式:
📅 2024-03-04 还款提醒
━━━━━━━━━━━━━━━━
🔴 今日应还 (3件)
  • JD-2024-001 | 张三 | ¥5,000.00
  • JD-2024-003 | 李四 | ¥3,000.00
  ...

🟡 明日应还 (2件)
  • JD-2024-005 | 王五 | ¥2,000.00
  ...

⚠️ 已逾期 (1件)
  • JD-2024-002 | 赵六 | ¥10,000.00 (逾期3天)
━━━━━━━━━━━━━━━━
[查看全部案件]
```

## 三、资产包运营Tab页

### 功能模块

#### 3.1 资产包概览
```
┌─────────────────────────────────────────────────────────────┐
│ 资产包: 2024-Q1-Pack-001                                    │
├─────────────────────────────────────────────────────────────┤
│ 总案件数: 150    总金额: ¥5,000,000                         │
├──────────────┬──────────────┬──────────────┬───────────────┤
│   应还金额   │   实还金额   │   逾期金额   │   回收率      │
│ ¥2,500,000   │ ¥1,800,000   │ ¥  500,000   │    72%        │
└──────────────┴──────────────┴──────────────┴───────────────┘
```

#### 3.2 图表分析
- **饼图**: 还款状态分布（正常/逾期/结清）
- **折线图**: 时间序列 - 月度应还vs实还趋势
- **柱状图**: 资产包对比 - 多个包回收率对比
- **热力图**: 逾期天数分布

#### 3.3 多维度分析
- 时间维度: 日/周/月/季度/年
- 案件维度: 按类型/地区/金额区间
- 对比维度: 资产包A vs 资产包B

## 四、底表上传自动维护

### 功能描述
- 上传标准格式Excel底表
- 系统自动解析并创建/更新资产包
- 支持字段映射配置

### 底表模板字段
```
| 案件编号 | 客户姓名 | 身份证号 | 手机号 | 本金金额 | 利息金额 | 起诉日期 | 资产包名称 |
```

### 处理流程
```
上传Excel → 字段映射 → 数据校验 → 导入资产包 → 生成案件 → 关联还款计划
```

## 五、OCR/文字提取自动填充

### 支持输入
- 图片上传（调解书、判决书、身份证等）
- 文本粘贴（从其他地方复制的案件信息）

### 识别字段
```
图片类型: 调解书
提取字段:
  - 案号
  - 当事人姓名
  - 身份证号
  - 调解金额
  - 还款期限
  - 签署日期

图片类型: 判决书
提取字段:
  - 案号
  - 判决金额
  - 判决日期
  - 生效日期
```

### 实现方案
- 后端集成OCR服务（如：百度OCR、腾讯云OCR）
- 或使用本地OCR库（如：Tesseract）
- 前端提供图片上传和预览
