## 一、案件生命周期流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           案件生命周期流程图                                   │
└─────────────────────────────────────────────────────────────────────────────┘

【阶段1: 待处理】
    │
    │ 录入案件基本信息
    ▼
【阶段2: 已立案】
    │
    │ 录入立案信息、分配法官
    ▼
【阶段3: 保全中】
    │
    │ 进行诉讼保全（冻结财产）
    │ 或：已协商无需冻结
    ▼
【阶段4: 分流处理】
    │
    ├──────────────────────────┬──────────────────────────┐
    │                          │                          │
    ▼                          ▼                          ▼
【调解结案】              【判决结案】              【其他结案】
    │                          │                          │
    │ 调解员组织调解           │ 开庭审理                │ 撤诉等
    │ 达成调解协议             │ 出具判决书              │
    │                          │                          │
    ▼                          ▼                          ▼
【阶段5: 已结案-履约监督中】
    │
    │ 监督债务人按调解书/判决书履约
    │ 记录还款情况
    │
    ├──────────────────────────┬──────────────────────────┐
    │                          │                          │
    ▼                          ▼                          ▼
【履约完成】              【不完全履约】            【完全不履约】
    │                          │                          │
    │ 未履约金额 = 0           │ 已还部分金额            │ 未还任何金额
    │ 案件正常结束             │ 继续监督或申请执行      │ 申请强制执行
    │                          │                          │
    ▼                          │                          ▼
【阶段6: 案件结束】           │                    【阶段6: 执行中】
                               │                          │
                               │                    【执行是保障履约的手段】
                               │                          │
                               ▼          ┌───────────────┴───────────────┐
                         【继续还款】   【执行回款】                【终结执行/终本】
                               │          │                               │
                               │          │ 通过法院强制划扣/           │ 暂无财产可供执行
                               │          │ 促使债务人主动还款          │ 等待发现新线索
                               │          │                               │
                               ▼          ▼                               ▼
                         【未履约金额=0】 【未履约金额=0】          【执恢立案】
                               │          │                               │
                               └──────────┴───────────────┬───────────────┘
                                                          │
                                                          ▼
                                                  【阶段7: 案件结束】
                                                          │
                                            【履约完成/债权实现是案件结束的唯一标志】
                                                          │
                                                          ▼
                                                  🏁 案件正常结束
```

## 二、案件状态定义

| 状态值 | 显示名称 | 说明 |
|--------|----------|------|
| pending | 待处理-待立案 | 刚录入，等待立案 |
| filing | 已立案-待保全 | 已分配案号，等待保全处理 |
| preservation | 保全中-待处理 | 保全进行中，等待调解/判决 |
| mediation | 调解结束-待履约 | 调解结案，等待履约监督 |
| judgment | 判决结束-待履约 | 判决结案，等待履约监督 |
| closed | 已结案-履约监督中 | 结案后监督履约阶段 |
| execution | 执行中 | **强制执行阶段** - 保障履约的手段 |
| finished | 案件结束 | **履约完成** - 未履约金额=0 |

### 关键理解

```
┌────────────────────────────────────────────────────────────┐
│  💡 核心概念：执行 ≠ 案件结束                               │
│                                                            │
│  • 执行是保障债务人履约的手段                              │
│  • 案件结束的唯一标志：未履约金额 = 0（履约完成）          │
│                                                            │
│  状态流转：                                                │
│  调解/判决结案 → 履约监督中 → 未履约金额=0 → 案件结束     │
│                              ↓                             │
│                        未履约金额>0                       │
│                              ↓                             │
│                        申请强制执行 → 执行回款 → 继续监督   │
└────────────────────────────────────────────────────────────┘
```

## 四、关键计算逻辑

### 4.1 金额计算

| 计算项 | 公式 | 适用场景 |
|--------|------|----------|
| 调解结果金额 | = 订单金额 + 订单外协商金额 | 调解结案时 |
| 判决结果金额 | = 判决书确定金额 | 判决结案时 |
| 结案结果金额 | = 调解结果金额 或 判决结果金额 | 通用 |
| 未履约金额 | = 结案结果金额 - 已还总额 | 履约监督时 |
| 执行标的金额 | 默认 = 未履约金额（可手动修改） | 申请执行时 |

### 4.2 状态自动判断

| 条件 | 自动设置状态 | 说明 |
|------|-------------|------|
| 录入案件基本信息 | → 待处理 | 初始状态 |
| 录入立案信息 | → 已立案 | 进入诉讼程序 |
| 录入执保信息 | → 保全中 | 财产保全阶段 |
| 录入调解/判决信息 | → 调解中/判决中 | 等待结案 |
| 录入结案日期 | → 已结案-履约监督中 | 开始监督履约 |
| 生成执行案件 | → 执行中 | **强制执行手段** |
| 未履约金额 = 0 | → 案件结束 | **唯一结束标志** |

### 4.3 执行与案件结束的关系

```
┌─────────────────────────────────────────────────────────────────────┐
│                     执行只是手段，履约完成才是目标                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   【已结案-履约监督中】                                             │
│          │                                                          │
│          ├── 债务人主动还款 ───────────────→ 【未履约金额=0】        │
│          │                                      │                   │
│          │                                      ▼                   │
│          │                              【案件结束】✅               │
│          │                                                          │
│          └── 债务人不履约 ───────→ 【生成执行案件】                  │
│                                              │                      │
│                                              ▼                      │
│                                      【执行中】（强制手段）          │
│                                              │                      │
│                          ┌───────────────────┼───────────────────┐  │
│                          │                   │                   │  │
│                          ▼                   ▼                   ▼  │
│                   【法院划扣】      【主动还款】      【终本/终结】  │
│                          │               │                       │  │
│                          │               │                       │  │
│                          └───┬───────────┘                       │  │
│                              ▼                                   │  │
│                      【执行回款计入已还总额】                      │  │
│                              │                                   │  │
│                              ▼                                   │  │
│                      【回到履约监督中】                           │  │
│                              │                                   │  │
│                              └──→ 未履约金额=0?                  │  │
│                                      │                           │  │
│                                      ├── 是 → 【案件结束】✅     │  │
│                                      │                           │  │
│                                      └── 否 → 继续执行/执恢      │  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 五、数据关系

### 5.1 案件关联关系

```
原案件（1）
    │
    ├─→ 执行案件 1（N）
    │       │
    │       └─→ 执恢记录（N，子流程）
    │
    ├─→ 执行案件 2（N）
    │
    └─→ ...
```

### 5.2 一对多关系

- 一个原案件 → 多个执行案件（可能多次申请执行）
- 一个执行案件 → 多个执恢记录（恢复执行子流程）
- 一个案件 → 多期还款明细

## 七、已确认事项 ✓

| # | 问题 | 确认结果 |
|---|------|----------|
| 1 | **字段必填项** | 阶段流转时该阶段字段需全部填写完成 |

| 5 | **执恢逻辑** | 执恢是执行案件的子流程 |

### 核心理解更新（重要！）

```diff
- 之前理解：执行完毕 = 案件结束
+ 正确理解：履约完成 = 案件结束

  执行只是保障债务人履约的强制手段
  ├── 执行回款会增加"已还总额"
  ├── 已还总额增加会减少"未履约金额"
  └── 未履约金额=0时，案件才真正结束
```
